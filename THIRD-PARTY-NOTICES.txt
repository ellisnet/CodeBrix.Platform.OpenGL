CodeBrix.Platform.OpenGL is a .NET 10 port of the Silk.NET.OpenGL v2.23.0
NuGet package and its immediate dependencies (Silk.NET.Core and
Silk.NET.Maths). The port incorporates substantial third-party source
code from the upstream Silk.NET project, adapted to build against .NET 10
without the Silk.NET.SilkTouch Roslyn source generator.

Upstream test projects ported into tests/CodeBrix.Platform.OpenGL.Tests/:

    src/Core/Silk.NET.Core.Tests   -> tests/CodeBrix.Platform.OpenGL.Tests/Core/
    src/Maths/Silk.NET.Maths.Tests -> tests/CodeBrix.Platform.OpenGL.Tests/Maths/

(Silk.NET has no Silk.NET.OpenGL.Tests upstream project; upstream validates
OpenGL via sample applications that require a GL context, not via xUnit.)

Modifications during port:

    - Target framework narrowed from the multi-TFM upstream set to net10.0 only.
    - All `#if` framework-selection conditional-compile blocks for older
      frameworks (netstandard2.0, netstandard2.1, netcoreapp3.1, net5.0,
      net6.0, net7.0-android, net7.0-ios) were resolved at port time: the
      .NET-10-applicable branch was kept, the rest stripped.
    - The Silk.NET.SilkTouch Roslyn source generator was NOT ported.
      Instead, the single large generated file that SilkTouch emits for
      Silk.NET.OpenGL (~150k lines containing ~3,680 method body
      implementations and the GeneratedVTable class) was captured from an
      upstream build and is committed as static source at
      src/CodeBrix.Platform.OpenGL/OpenGL/GL.gen.bodies.cs.
    - Upstream namespaces were renamed:
        Silk.NET.OpenGL  -> CodeBrix.Platform.OpenGL
        Silk.NET.Core    -> CodeBrix.Platform.OpenGL.Core
        Silk.NET.Maths   -> CodeBrix.Platform.OpenGL.Maths
      Every ported file's namespace line carries a
      `//was previously: <upstream-ns>;` provenance comment.
    - The `Ultz.Bcl.Half` NuGet polyfill dependency was dropped; .NET 10
      provides `System.Half` natively.
    - The `Silk.NET` public-API-diff discipline (`SilkPublicApiExempt`
      property, `PublicAPI.Shipped.txt`/`.Unshipped.txt` files, and the
      `Microsoft.CodeAnalysis.PublicApiAnalyzers` dependency) was removed
      — this fork does not maintain that discipline.
    - The `RequiresLocationAttribute` polyfill was dropped in favor of the
      BCL `System.Runtime.CompilerServices.RequiresLocationAttribute`
      available in .NET 10.
    - Tests were migrated from xUnit v2 to xUnit v3, including the
      mandatory `TestContext.Current.CancellationToken` threading for any
      cancellable API call.
    - Upstream file headers ("Licensed to the .NET Foundation under one
      or more agreements ...") are preserved verbatim on every ported
      file per skill convention.
